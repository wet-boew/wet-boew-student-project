Contributor Guidelines

Everyone is welcome to contribute to WET! A diverse community of contributors produces better quality results.

The Web Experience Toolkit Project is managed openly on GitHub, this allows for discussion through the issues tracker. This encourages a free ow of ideas. Each repository has an issue tracker. Post your issue in the repository associated to the code base with which you are working. If you're unsure, post it to "WET-BOEW" repository on GitHub which is our primary repository.

A multi-level review process (automated and manual reviews) for contributions ensures code integrity.

How to Contribute:

  • Pull Requests

    Request that we add your changes.

  • Issues and suggestions

    Tell us about a bug or suggest a feature or improvement.

  • Documentation

    Create documentation that explains how to implement a plug-in or polyﬁll.

  • Usability heuristic review

    Conduct a heuristic review on a WET component.

  • Web Guides

    Suggest edits and improvements to the guidelines.

  • Testing

    How to test your code before you submit it to the project.
